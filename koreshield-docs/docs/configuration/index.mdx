# Settings & Policies

KoreShield is configured from the **KoreShield Dashboard** for hosted customers. Enterprise customers running a self-hosted deployment can apply the same settings using environment variables or a managed configuration bundle provided by KoreShield.

## What You Can Configure

- Security sensitivity and default action
- Policy rules for prompt injection, data loss, and abuse
- Provider connections (OpenAI, Anthropic, DeepSeek, Gemini, Azure OpenAI)
- Rate limits and usage guardrails
- Authentication modes (JWT, API key)

## Where You Configure It

- **Hosted:** Use the Dashboard policy and settings pages.
- **Self-hosted:** Apply settings via your deployment configuration and environment variables.

If you need help enabling a capability or you’re unsure where to set it, contact support at support@koreshield.com.
