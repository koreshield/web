# Compliance Guide

KoreShield provides security controls and audit data you can map into your compliance program.

This page is technical guidance, not legal advice.

## What The Codebase Provides Today

- Authenticated management APIs (`/v1/management/*`)
- JWT validation with strict `iss`/`aud` checks
- API-key support with revocation/expiry checks
- Request/security logging pipeline
- Policy/rule management and audit report endpoints

## Controls You Can Implement

- Access control: use role-gated endpoints and least privilege.
- Audit trail: retain request and security logs from reports/log APIs.
- Data minimization: avoid sending unnecessary PII in prompts.
- Incident response: monitor and alert on blocked or suspicious activity.

## Evidence Collection Suggestions

- Export report data via `/v1/reports*`.
- Track auth failures and API key misuse attempts from logs.
- Record deployment config and JWT settings at release time.

## Important Scope Note

KoreShield does not automatically grant certifications. Your organization must implement the required operational controls and complete formal audits.
