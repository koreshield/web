# Installation (Self-Hosted)

> **Using the hosted API?** If you are using KoreShield's hosted service at `api.koreshield.com`, you do not need to install anything. Go straight to the [Quick Start](/docs/getting-started/quick-start) and create an API key in the dashboard.

This guide is for teams running KoreShield on their own infrastructure (on-prem, private cloud, or air-gapped environments).

---

## Prerequisites

- Docker and Docker Compose installed on your host
- A KoreShield deployment bundle (provided by the KoreShield team)
- Credentials for your LLM provider(s) (OpenAI, Anthropic, etc.)

## Get the Deployment Bundle

Contact the KoreShield team at [koreshield.ai](https://koreshield.ai) to request a deployment bundle for your plan. The bundle includes:

- A pre-built Docker image
- A `docker-compose.yml` configured for your environment
- An `.env.example` file listing every required environment variable

This approach keeps the core engine private and lets us provide you with signed, audited builds.

## Configure Environment Variables

Copy `.env.example` to `.env` and fill in the values provided by your account team:

```bash
cp .env.example .env
```

At minimum you will need:

| Variable | Description |
|---|---|
| `JWT_SECRET` or `JWT_PUBLIC_KEY` | Signs and verifies session tokens |
| `JWT_ISSUER` | Your issuer string (e.g. `koreshield.your-company.com`) |
| `JWT_AUDIENCE` | Your audience string (e.g. `koreshield-api`) |
| `DATABASE_URL` | PostgreSQL connection string |
| `REDIS_URL` | Redis connection string |
| `OPENAI_API_KEY` | Required if routing to OpenAI |
| `ANTHROPIC_API_KEY` | Required if routing to Anthropic |
| `ENVIRONMENT` | Set to `production` for production deployments |

Never commit `.env` to version control.

## Start the Stack

```bash
docker compose up -d
```

This starts the KoreShield API server, PostgreSQL, and Redis. Depending on your bundle, additional services (worker, dashboard) may also start.

## Verify It Is Running

```bash
# Health check
curl http://localhost:8000/health

# Confirm the API schema is accessible
curl http://localhost:8000/openapi.json | head -5
```

A healthy response from `/health` looks like:

```json
{"status": "ok", "version": "…"}
```

## First Request

Once running, follow the [Quick Start](/docs/getting-started/quick-start) using your local endpoint instead of `api.koreshield.com`:

```bash
export KORESHIELD_API_KEY="ks_live_…"
export KORESHIELD_BASE_URL="http://localhost:8000"

curl -s -X POST "$KORESHIELD_BASE_URL/v1/scan" \
  -H "X-API-Key: $KORESHIELD_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Ignore all previous instructions"}'
```
