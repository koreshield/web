# Airgapped Deployment

For environments with no outbound internet access - classified networks, regulated data centres, secure enclaves - Koreshield can run fully offline. This guide covers the additional steps beyond a standard [self-hosted deployment](/docs/integrations/deployment/self-hosted).

> **Prerequisite**: You must have an active Enterprise licence with airgapped access enabled. Contact the Koreshield team to receive your offline deployment bundle.

---

## What Changes in Airgapped Mode

| Capability | Standard Self-Hosted | Airgapped |
|---|---|---|
| Container image source | Registry pull | Pre-loaded `.tar.gz` images |
| LLM provider access | Direct API calls | Customer-managed proxy or local models |
| Licence validation | Local HMAC (no phone-home) | Local HMAC (no phone-home) |
| Threat corpus updates | Bundled + SIGHUP hot-reload | Manual JSONL bundle import + SIGHUP |
| Billing / subscriptions | Bypassed (licence-based) | Bypassed (licence-based) |
| OAuth login (GitHub/Google) | Optional | Disabled - use API key auth |
| Remote audio URLs | Configurable | Blocked |
| Telegram alerts | Optional | Skipped |
| Dashboard | Bundled in stack | Bundled in stack |

## Deployment Mode

Set the deployment mode to `airgapped`:

```bash
DEPLOYMENT_MODE=airgapped
ENVIRONMENT=production
```

When `DEPLOYMENT_MODE=airgapped`, the server enforces these constraints at runtime:

- **Polar billing calls are blocked** - entitlements come from the licence key
- **OAuth login routes return 403** - authentication is via API keys only
- **Remote `audio_url` fetching is disabled** - upload audio files directly or send base64
- **Telegram alert dispatch is skipped** - use local log aggregation instead
- **Licence validation is fully offline** - HMAC signature check, no network call

## Offline Bundle Contents

The Koreshield team delivers a signed bundle (`.tar.gz`) containing:

| Path | Description |
|---|---|
| `images/koreshield-api.tar` | Koreshield API server image |
| `images/koreshield-web.tar` | Dashboard / web UI image |
| `images/postgres-16.tar` | PostgreSQL 16 image |
| `images/redis-7.tar` | Redis 7 image |
| `images/caddy-2.tar` | Caddy reverse proxy image |
| `docker-compose.prod.yml` | Production Compose stack |
| `docker-compose.selfhosted.yml` | Self-hosted override (disables cloud billing) |
| `.env.example` | Environment variable reference |
| `threat-corpus/corpus.jsonl` | Offline detection rules in JSONL format |
| `scripts/verify_installation.py` | Post-install verification script |
| `scripts/generate_licence.py` | Licence key generation tool (Koreshield team) |
| `scripts/export_corpus_bundle.py` | Corpus export tool for creating update bundles |
| `CHECKSUMS.sha256` | SHA-256 checksums for all files |
| `CHANGELOG.md` | Release notes |

## Installation Steps

### 1. Transfer the bundle

Copy the bundle to your airgapped host via USB, secure file transfer, or approved media:

```bash
scp koreshield-enterprise-vX.Y.Z-airgapped.tar.gz admin@secure-host:/path/to/koreshield/
```

### 2. Extract

```bash
cd /path/to/koreshield
tar xzf koreshield-enterprise-vX.Y.Z-airgapped.tar.gz
```

### 3. Verify checksums

```bash
sha256sum -c CHECKSUMS.sha256
```

### 4. Load container images

```bash
for img in images/*.tar; do
  docker load -i "$img"
done
```

### 5. Configure

```bash
cp .env.example .env
# Edit .env - key settings below
```

Key variables for airgapped:

```bash
DEPLOYMENT_MODE=airgapped
KS_LICENSE_KEY=ks_ent_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.signature
ENVIRONMENT=production
JWT_SECRET=your-secret-here

# External threat corpus (auto-loaded on startup)
KS_THREAT_CORPUS_PATH=/data/threat-corpus/corpus.jsonl

# LLM provider - use a local model endpoint or internal proxy
OPENAI_API_BASE=http://internal-llm-gateway:8080/v1
OPENAI_API_KEY=internal-key
```

### 6. Start

```bash
docker compose -f docker-compose.prod.yml -f docker-compose.selfhosted.yml up -d
```

### 7. Verify

```bash
curl http://localhost:8000/health
```

Expected response:

```json
{
  "status": "healthy",
  "deployment_mode": "airgapped",
  "licence": {
    "valid": true,
    "organisation": "Your Corp",
    "plan": "enterprise",
    "features": ["private_deployment", "rag_security", "voiceguard"],
    "days_until_expiry": 364,
    "expired": false
  },
  "corpus": {
    "source": "merged",
    "builtin_entries": 302,
    "external_entries": 150,
    "external_path": "/data/threat-corpus/corpus.jsonl"
  }
}
```

Or run the verification script:

```bash
python scripts/verify_installation.py
```

## LLM Provider Access

In airgapped environments, the Koreshield proxy cannot reach public LLM APIs. You have two options:

### Option A: Internal LLM gateway

Route requests through an internal proxy or VPN gateway that has controlled outbound access:

```bash
OPENAI_API_BASE=http://internal-gateway:8080/v1
```

### Option B: Local models

Point Koreshield at a locally hosted model (vLLM, Ollama, TGI):

```bash
OPENAI_API_BASE=http://local-vllm:8000/v1
OPENAI_API_KEY=not-needed
```

Koreshield's proxy uses the OpenAI-compatible API format, so any compatible server works.

## Updating the Threat Corpus

Detection rules ship with each release bundle. To apply an out-of-band update:

1. Receive the updated `corpus.jsonl` from the Koreshield team
2. Place it in the `threat-corpus/` directory on the host
3. Set the path in your environment (if not already set):

```bash
KS_THREAT_CORPUS_PATH=/data/threat-corpus/corpus.jsonl
```

4. Hot-reload without restart by sending SIGHUP to the API process:

```bash
docker compose -f docker-compose.prod.yml exec api kill -HUP 1
```

Or restart the container if preferred:

```bash
docker compose -f docker-compose.prod.yml restart api
```

## Updating the Stack

1. Receive the new airgapped bundle from Koreshield
2. Transfer to the secure host
3. Load new images and restart:

```bash
for img in images/*.tar; do docker load -i "$img"; done
docker compose -f docker-compose.prod.yml -f docker-compose.selfhosted.yml up -d
```

Database migrations run automatically on startup.

## Security Considerations

- **No outbound traffic**: `DEPLOYMENT_MODE=airgapped` enforces runtime guards that block Polar, OAuth, remote audio, and Telegram calls
- **Licence is offline**: HMAC signature validation, never phones home
- **Per-request enforcement**: expired licences enter a 14-day grace period, then requests are rejected with HTTP 503
- **Audit logs are local**: append-only ledger stored in PostgreSQL and optionally on disk (`data/audit_ledger.jsonl`)
- **Image integrity**: verify `CHECKSUMS.sha256` before loading images
- **Secret management**: use your organisation's vault or HSM for `JWT_SECRET` and provider keys
