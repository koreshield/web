# Self-Hosted Deployment

Self-hosted deployment is available on the **Enterprise plan**. You run Koreshield inside your own infrastructure - VPC, on-prem data centre, or private cloud - with full control over data residency, network policies, and update cadence.

> **Not on Enterprise?** Use the hosted API at `api.koreshield.com` instead. See the [Quick Start](/docs/getting-started/quick-start).

---

## What You Receive

When you activate a self-hosted Enterprise licence, the Koreshield team provides:

| Deliverable | Description |
|---|---|
| Docker image bundle | Signed, audited container images for the API server, status updater, and web dashboard |
| `docker-compose.prod.yml` | Compose stack with Postgres, Redis, and Caddy |
| `docker-compose.selfhosted.yml` | Self-hosted override that disables cloud billing and sets correct env vars |
| `.env.example` | Full environment variable reference pre-filled with safe defaults |
| Licence key | A `KS_LICENSE_KEY` tied to your organisation, plan limits, and expiry date |
| Threat corpus snapshot | Offline-compatible JSONL detection rules for airgapped environments |

## Architecture

Everything runs inside your network boundary. No data leaves your infrastructure.

| Service | Role | Port |
|---|---|---|
| **Caddy** | Reverse proxy, TLS termination, security headers | 443 → 8000 |
| **Koreshield API** | Core security engine - scanning, detection, policy enforcement | 8000 |
| **PostgreSQL** | Primary datastore with per-tenant schema isolation | 5432 |
| **Redis** | Session cache, rate limiting, real-time pub/sub | 6379 |

**Traffic flow**: Caddy → Koreshield API → PostgreSQL / Redis. All inter-service communication is on an internal Docker network. External traffic only enters through Caddy.

## Quick Start

### 1. Set the deployment mode and licence

Create a `.env` file (start from `.env.example`) and configure:

```bash
DEPLOYMENT_MODE=self-hosted
KS_LICENSE_KEY=ks_ent_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.signature
ENVIRONMENT=production
```

The licence key is validated on startup - the server will not start without a valid Enterprise licence.

### 2. Configure core secrets

```bash
JWT_SECRET=replace-with-a-strong-random-secret
JWT_ISSUER=koreshield.yourcompany.com
JWT_AUDIENCE=koreshield-api
DATABASE_URL=postgresql+asyncpg://koreshield:secret@postgres:5432/koreshield
REDIS_URL=redis://redis:6379/0
```

### 3. Add LLM provider keys

Add keys for whichever providers your teams use:

```bash
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
DEEPSEEK_API_KEY=sk-...
```

### 4. Start the stack

Use the self-hosted override to disable cloud billing automatically:

```bash
docker compose -f docker-compose.prod.yml -f docker-compose.selfhosted.yml up -d
```

### 5. Verify

```bash
curl http://localhost:8000/health
```

A healthy response includes `deployment_mode`, licence status, and corpus metadata:

```json
{
  "status": "healthy",
  "deployment_mode": "self-hosted",
  "licence": {
    "valid": true,
    "organisation": "Your Corp",
    "plan": "enterprise",
    "features": ["private_deployment", "rag_security", "voiceguard", "..."],
    "days_until_expiry": 364,
    "expired": false,
    "in_grace_period": false,
    "monthly_usage": 0,
    "monthly_cap": null,
    "requests_remaining": null
  },
  "corpus": {
    "source": "built-in",
    "builtin_entries": 302
  }
}
```

You can also run the verification script:

```bash
python scripts/verify_installation.py
```

## How It Works

### Licence Validation

- **Startup check**: the server validates `KS_LICENSE_KEY` and refuses to boot if invalid
- **Per-request enforcement**: expired licences enter a **14-day grace period** where the server continues running with `X-KoreShield-Licence: expiring` headers. After the grace period, requests receive HTTP 503
- **Monthly cap**: if your licence includes a `max_requests_per_month`, the server returns HTTP 429 when the cap is reached
- **Live reload**: send `SIGHUP` to the process to reload the licence key without restarting
- **No phone-home**: validation uses HMAC signatures - no network call is made

### Billing Bypass

Self-hosted deployments skip Polar (cloud billing) entirely. Entitlements are derived from the licence key's feature list and plan tier. All Enterprise features are unlocked by default.

### External Threat Corpus

You can provide an external JSONL corpus file for custom detection rules:

```bash
KS_THREAT_CORPUS_PATH=/data/threat-corpus/corpus.jsonl
```

Entries are merged with the built-in corpus on startup. Send `SIGHUP` to hot-reload the corpus without restarting. Use the export script to create a corpus bundle:

```bash
python scripts/export_corpus_bundle.py -o corpus-bundle.jsonl
```

## Tenant Data Isolation

Each tenant receives a dedicated PostgreSQL schema (`tenant_<id>`). Cross-tenant queries are prevented at the middleware layer. Audit logs are append-only and tagged with compliance labels (GDPR, SOC 2, HIPAA) based on tenant configuration.

See [RBAC](/docs/features/rbac) for role-based access control within a deployment.

## Monitoring

The production Compose stack includes:

- **Structured JSON logs** via `structlog` - compatible with any log aggregator (ELK, Datadog, Splunk)
- **Prometheus metrics** at `/metrics` - scrape every 15 seconds
- **`/health` endpoint** with licence status, corpus metadata, and service health

## Updating

1. The Koreshield team provides updated container images and a changelog
2. Pull the new images and restart:

```bash
docker compose -f docker-compose.prod.yml -f docker-compose.selfhosted.yml pull
docker compose -f docker-compose.prod.yml -f docker-compose.selfhosted.yml up -d
```

3. Database migrations run automatically on container startup via Alembic

For environments without internet access, see the [Airgapped Deployment](/docs/integrations/deployment/airgapped) guide.
