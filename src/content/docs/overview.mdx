# Welcome to KoreShield

KoreShield is an LLM security gateway. It sits between your application and your AI model provider, scanning every prompt and retrieved document for threats before they reach the model and before any damage is done.

## Why KoreShield

When you wire an LLM into your product, your users control the input. That means attackers can:

- **Inject instructions** into prompts to override your system prompt or extract internal data
- **Hide malicious commands** inside documents your RAG pipeline retrieves and passes to the model
- **Jailbreak the model** to bypass your content policies
- **Abuse your LLM API budget** with expensive or automated queries

KoreShield blocks all of this at the gateway layer, without requiring changes to your model provider or your prompts.

## How It Works

```
Your App  →  KoreShield  →  OpenAI / Anthropic / Gemini / DeepSeek
                ↓
         Scan + Policy
         Decision here
```

Every request passes through KoreShield first. It scans the prompt, applies your security policy, and either forwards the request to the model or blocks it with a structured error your app can handle. In most integrations, you only change the `base_url`. Your existing LLM code stays the same.

## What It Detects

- **Prompt injection:** direct attacks in user messages ("ignore previous instructions…")
- **Indirect / RAG injection:** malicious instructions hidden in retrieved documents, web pages, or emails the model reads
- **Jailbreaks:** persona injection, role-play bypasses, many-shot attacks, encoded requests
- **Policy violations:** content your organisation defines as off-limits (topics, formats, data types)
- **System prompt extraction:** attempts to make the model reveal its own instructions

## Supported Model Providers

KoreShield routes to the right provider automatically based on the model name you pass:

| Model prefix | Provider |
|---|---|
| `gpt-*`, `o1-*`, `o3-*` | OpenAI |
| `claude-*` | Anthropic |
| `gemini-*` | Google Gemini |
| `deepseek-*` | DeepSeek |
| `gpt-*` via Azure endpoint | Azure OpenAI |

## Authentication: Which Method Should I Use?

KoreShield supports two auth methods. **Use API keys for almost everything.**

| Method | When to use |
|---|---|
| `X-API-Key: ks_live_…` | All SDK and direct API integrations. Create one in the dashboard. |
| `Authorization: Bearer <JWT>` | Server-to-server flows where you manage your own JWT issuance. Most users don't need this. |

You create an API key in the KoreShield dashboard. It works immediately, has no expiry by default, and can be scoped and rotated at any time.
